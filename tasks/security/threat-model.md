# 위협 모델

> 보안 세션(/project:security)이 관리한다.
> 프로젝트 아키텍처가 변경되면 이 문서를 업데이트한다.

## 시스템 구성

```
[사용자 브라우저] → HTTP:80 → [Docker 컨테이너 :3000] → [SQLite 파일]
                                       ↑
                                 EC2 t4g.small (폐쇄망)
```

## 신뢰 경계 (Trust Boundaries)
1. 클라이언트 ↔ 서버: HTTP 통신 (암호화 없음, 폐쇄망으로 보완)
2. 서버 ↔ DB: SQLite 파일 직접 접근 (서버 침해 시 DB 전체 노출)
3. EC2 인스턴스 ↔ 관리자: AWS SSM 접근 (권한 탈취 시 전체 장악)

## 위협 목록

### T-001: 인증 우회
- 심각도: Low (폐쇄망으로 위험 감소)
- 공격 벡터: 세션 쿠키 위조, 인증 없는 API 접근
- 현재 대응: 세션 쿠키 + bcryptjs 해싱
- 상태: 🟡 수용 (HTTPS 전환 시 해결)
- 근거: ISO 27001 A.9.4, ISMS-P 2.5

### T-002: SQL Injection
- 심각도: Medium
- 공격 벡터: 사용자 입력을 통한 DB 조작
- 현재 대응: Prisma ORM 파라미터 바인딩
- 상태: 🟢 대응됨
- 근거: ISO 27001 A.14.2, ISMS-P 2.8

### T-003: DB 파일 직접 접근
- 심각도: Medium
- 공격 벡터: 서버 접근 시 DB 파일 직접 복사/수정
- 현재 대응: SSM 접근 제어
- 상태: 🟡 개선 필요 (파일 권한, 백업 암호화)
- 근거: ISO 27001 A.12.3, ISMS-P 2.9

### T-004: Docker 이미지 내 민감 정보
- 심각도: High
- 공격 벡터: 이미지에 소스코드, 시크릿 포함
- 현재 대응: (확인 필요)
- 상태: 🔴 미검토
- 근거: ISO 27001 A.12.6, ISMS-P 2.10

### T-005: XSS
- 심각도: Medium
- 공격 벡터: 사용자 입력이 HTML로 렌더링
- 현재 대응: React 기본 이스케이핑
- 상태: 🟡 확인 필요
- 근거: ISO 27001 A.14.2, ISMS-P 2.8

### T-006: 개인정보 비보호 (PII Exposure)
- 심각도: Critical
- 공격 벡터: Employee 개인정보 평문 저장, 접근 로그 미비
- 현재 대응: (확인 필요)
- 상태: 🔴 미검토
- 근거: ISO 27701 7.4, ISMS-P 3.1~3.5

### T-007: 감사 로그 미비
- 심각도: High
- 공격 벡터: 보안 사고 시 추적 불가, 내부자 위협 감지 불가
- 현재 대응: AuditLog 모델 존재, 전면 적용 여부 확인 필요
- 상태: 🟡 확인 필요
- 근거: ISO 27001 A.12.4, ISMS-P 2.11

### T-008: 세션 관리 미흡
- 심각도: Medium
- 공격 벡터: 세션 무기한 유지, 동시 세션 미제한
- 현재 대응: (확인 필요)
- 상태: 🟡 확인 필요
- 근거: ISO 27001 A.9.4, ISMS-P 2.5
