## 배경 (백로그 F2)
- "구글 드라이브가 디폴트가 아닌 다운로드를 디폴트로": 진단 결과 설계 자체가 GDrive 단일 경로 —
  다운로드 경로가 아예 없고, GDrive 미설정이면 xlsx 생성조차 분기 안에서 건너뜀.
- "어떤 증적이 생성되는지 미리보기": 데이터(ArchiveData.recordCount)와 상세 GET API 는 이미
  존재하나 UI 미연결.

## 구현
- lib/archive-export.ts: 기존 export 라우트의 시트 구성(요약/라이선스/조직원/변경이력)을
  buildArchiveWorkbook 으로 추출 — 다운로드/업로드가 단일 출처 공유 (동작 불변).
- GET /api/admin/archives/[id]/export: Content-Disposition attachment 직접 다운로드.
  requireAdmin 자체 가드(proxy.ts 가 GET 을 비인증 통과시키므로 필수 — lessons 준수).
  다운로드 시 ArchiveLog info 기록(누가 받았는지 — 증적 추적).
- UI: 목록의 COMPLETED 행에 다운로드 버튼(파란색, 기본) + GDrive 업로드(회색, 보조) 병행.
  상세 패널에 "미리보기" 섹션 — dataType별 건수 (i18n: 기존 nav 라벨 재사용 + preview/assets 신규).
