## 정책 (dev-022 / dev-024)
- 내용연수(usefulLifeYears)·감가상각 정보는 SUPER_ADMIN(isSuperAdmin) 또는 LIFECYCLE_VISIBLE_TO_USER
  플래그가 켜진 경우에만 노출. 그 외 사용자에게는 화면(dev-024)뿐 아니라 API 페이로드에서도 비노출.

## 사용자 확정 (질문 응답)
- ① 마스킹 범위 = "내용연수·감가상각만". usefulLifeYears + 감가상각 파생(billingCycle=DEPRECIATION 일 때
  monthlyCost)만 제거. cost/purchaseDate/expiryDate 는 유지(여러 비-라이프사이클 표면이 사용).
- ② 리포트/내보내기(대시보드·월별·export/all)는 이번 제외 → 후속 티켓(dev-026 초안).

## 설계 결정
- 판정 헬퍼 isLifecycleVisible(user): isSuperAdmin || getFeatureFlag("LIFECYCLE_VISIBLE_TO_USER").
  미인증(null)은 fail-closed(비노출). feature-flags 라우트도 이 헬퍼로 단일화.
- 마스킹 헬퍼 maskAssetLifecycle(asset, visible): visible=false 면 hardwareDetail.usefulLifeYears=null,
  billingCycle==="DEPRECIATION" 이면 monthlyCost=null. cost/날짜는 손대지 않음.
- 마스킹 대상은 본질적으로 하드웨어(PC) 자산만 — 비-하드웨어는 hardwareDetail 없음 + DEPRECIATION 아님 → no-op.
  따라서 도메인/클라우드/계약 페이지는 영향 0.
- PUT 데이터 손실 방지: 편집 폼은 마스킹된 GET 으로 채워지므로 usefulLifeYears 가 빈값→기본 5 로 전송될 수 있음.
  비-권한자면 PUT 진입부에서 body.hardwareDetail.usefulLifeYears 를 DB 기존값으로 강제 치환(heal)하여
  덮어쓰기를 차단. cost/날짜는 마스킹되지 않으므로 폼에 실제값이 있어 그대로 보존됨(가드 불필요).
- GET 엔드포인트는 현재 라우트 인증이 없음(미들웨어 부재). 마스킹용 getCurrentUser() 만 추가하고
  401 게이트는 추가하지 않음(범위 외·기존 동작 보존). 미인증은 fail-closed 마스킹으로 처리.
