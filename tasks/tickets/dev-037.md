## 배경 (백로그 F5 — "실제 동작하는 것이 맞는지" 답변: 부분 동작)
정의 계층(관리 CRUD)·API(검증 포함)는 완성인데 소비 계층이 0 — 자산 폼에 분류 선택 UI 가 없어
subCategoryId 가 채워질 경로가 없었음(시드/직접 API 호출 제외). 유일 조회 API 도 requireAdmin
이라 일반 폼에서 트리를 불러올 수 없었음.

## 구현
- 공용 GET /api/asset-classifications: 로그인 사용자, getCurrentUser 자체 가드(proxy GET 통과 대응).
- ClassificationSelect: 트리 자체 fetch, 대분류는 value(소분류)에서 파생(역추적) — effect 내
  setState 없이 편집 초기화 해결. 분류 미설정이면 렌더 안 함.
- 폼 8곳: CiaScoreInput 직전 배치 + payload subCategoryId (POST/PUT 은 기존 검증 로직이 이미 수용).
  편집 로더에서 d.subCategoryId 초기화.
- 목록 4곳: 이름 셀에 보라색 소분류 배지(title=대분류) — 컬럼 추가 없음(dev-035 패턴).

## 보류 (후속)
- 목록 분류 "필터" 드롭다운(API 파라미터는 이미 존재) — 4개 목록 필터 행 확장은 별도.
- ISMS 대상 플래그(isIsmsTarget)의 리포트/내보내기 반영.
