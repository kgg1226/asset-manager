## 배경
대시보드 월간 비용이 월별 구독에서 한 사이클 뒤 사라지는 문제. 근본 원인:
신규 폼이 자동계산 "갱신일"(calcRenewalDate)을 hidden input 으로 **expiryDate 에 저장** →
대시보드/큐가 expiryDate 를 "만료"로 보고 한 주기 뒤 INACTIVE 처리.
스키마엔 이미 expiryDate(만료)/renewalDate(갱신)/renewalDateManual/renewalStatus 분리 존재 →
혼용을 바로잡는 작업.

## 확정 모델 (사용자 합의)
- 만료일(expiryDate) = 실제 종료일. 기본 null(진행 중). "계약 종료" 액션으로만 설정.
- 갱신일(renewalDate) = 다음 갱신일(구매일+주기 자동). "갱신" 버튼으로 롤포워드.
- 월별 = 취소 전엔 진행. 생성 폼에 만료일 셀렉트 없음.
- 액션 버튼은 분리: 갱신 / 연장 / 계약 종료.

## 1단계 구현(본 티켓)
- A 생성(폼·액션·API): renewalDate 자동저장, expiryDate=null.
- C 갱신 큐: 유효 갱신일 기준 + 하위호환 expiryDate 별칭 반환(소비자 무수정).
- F 백필 스크립트(멱등): 잘못 저장된 expiryDate → renewalDate 이전.

## 후속
- 2단계: 갱신/연장/계약종료 버튼(갱신일 롤포워드, 만료일 등록) + 위젯 표시.
- 3단계: 알림 배치·대시보드 카드/차트 비용 기준 정렬.
- 배포: entrypoint 에 backfill 스크립트 추가 또는 1회 수동 실행(`DATABASE_URL=… node scripts/backfill-license-renewal.mjs`).
