## 배경 — 사용자 보고 (v1.62.0)
① A·B 자산에서 도착 지점이 같은 연결을 그리면 표시가 사라지고 탭 전환/새로고침에 재등장
② 연결 추가 후 새로고침 시 노드 배치가 뒤죽박죽
③ 새로고침하면 페이지 복원이 아니라 자산지도 홈(갤러리)으로 떨어짐

## 원인
① **엣지 in-place 변이(감사 P2 #4 실사고화)**: distributeEdgeHandles 가 인자 엣지의 핸들을 직접
   대입 변이. handleSaveLink 는 RF 스토어의 현재 엣지 객체를 그대로 넘기므로, 같은 도착변에 두 번째
   선이 붙어 기존 선 핸들이 재분배되는 순간 RF 내부 상태와 어긋나 두 선 모두 렌더에서 소실.
   재조회 시 새 객체로 재구성돼 되살아나는 것까지 증상과 일치.
③ **localStorage 동기화 레이스**: activeWorkspace 동기화 effect 가 마운트 첫 실행(null)에서
   removeItem → 비동기 init() 이 복원 키를 읽기 전에 삭제 → 새로고침이 항상 갤러리로.
② ③의 낙하 후 재진입 흐름에서 복원이 조용히 건너뛰면 dagre 재배치가 남고, 병합 저장이 그
   재배치를 덮어써 영구화 — ①·③ 해소가 근본이며, 복원 경로에 문자열(JSON) 저장본 방어를 추가.

## 수정
- distributeEdgeHandles **순수화**: 입력 복제 후 변이·복제본 반환. 호출부 2곳(fetchGraph·handleSaveLink)이
  반환 배열 사용 — RF 스토어/상태 객체 불변.
- localStorage 동기화 effect 첫 실행 스킵(초기 null 의 removeItem 차단) → 새로고침 페이지 복원 정상화.
- 위치 복원 effect 에 문자열 저장본 파싱 방어(복원 무산→덮어쓰기 영구화 차단).
