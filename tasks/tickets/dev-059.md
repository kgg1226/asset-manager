## 배경 — 사용자 보고 회귀 (화면 녹화)
"연결을 만들어도 전체 흐름도에 안 나옴 / 새로고침에 잠깐 보였다 사라짐 / 탭 갔다 오면 연결·자산 소실".

## 근본 원인 — 세 저장 규칙의 조합 (dev-058 단독 아님)
① 위치 저장이 all 뷰 전용(dev-049 가드 — 부분집합 덮어쓰기 방지책) → 비-all 뷰에서 배치한 자산 위치가
   영영 저장 안 됨 → 재조회 시 자산 소실.
② all 뷰 autosave 가 소속(visibleEdgeIds)을 "렌더된 엣지"로 재작성 → 엔드포인트 미배치라 안 그려진
   연결이 소속에서 삭제.
③ dev-058 이 소속 필터를 전 뷰 적용 → ②에서 깎인 연결이 모든 뷰에서 소실(이전엔 비-all 뷰에 잔존).

## 해법 — 덮어쓰기를 병합으로 (dev-049 가드의 근본 대체)
- **위치**: 저장본과 합집합 병합({...saved, ...canvas}) — 부분집합 캔버스는 자기 노드만 갱신·추가,
  나머지 보존 → **어느 뷰에서든 저장 안전**(all 뷰 전용 가드 자체가 불필요해짐). 캔버스에서 노드를
  제거하는 경로는 없어(추가 전용) 병합 부활 위험 없음.
- **소속**: autosave 재작성 전면 제거(passthrough). 변경은 명시 조작만 —
  생성 append(전 뷰, dev-056 확장) · 삭제 remove(신규) · 토글 set(기존).
- **섹션**: 재작성은 all 뷰 전용 유지(섹션 전체가 all 캔버스에서만 보임). 삭제/편집 경로 기존과 동일.
- beforeunload 도 동일 규칙 미러링.
