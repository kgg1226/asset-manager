## P0 — 데이터유실 (근본 원인)
flushSave가 PUT 응답을 검사하지 않음(await fetch 반환 무시). fetch는 4xx/5xx에 reject 안 함
→ 401(세션만료)/404(뷰삭제)/500(DB오류)에도 try가 throw 없이 끝나 savedState 병합 +
setSaveStatus("saved") 무조건 실행. dirtyRef=false 유지 → 후속 flushSave early-return →
변경분 영구 소실. beforeunload(dirtyRef 의존) 방어선도 무력화.
- 수정: `const res = await fetch(...)` + savedState 병합 전 `if (!res.ok) throw` → 실패 시
  catch가 setSaveStatus("dirty")+dirtyRef=true 복구, stale 병합 차단.

## P1-a — markDirty 누락 액션 (자동저장 안 됨 체감)
섹션 생성(handleAddSection)·편집 모달 이름/설명/색상 onBlur·onClick·삭제·자동정렬(handleAutoLayout)이
markDirty 미호출 → 다른 액션에 묻어가야만 저장. 각 지점 markDirty(300~500) 추가.

## P2-a — beforeunload edgeVisibility 누락
언로드 sendBeacon 페이로드에 edgeVisibility 없어 가시성 유실 가능 → flushSave와 동일 로직으로 포함.

## 저장 트리거 답 (사용자 질문)
하이브리드 — 액션이 markDirty 로 debounce 타이머를 건다(액션 단위 호출 + 시간 단위 저장).
노드드래그 2s, 엣지 300ms, 정렬 500ms, 뷰포트 3s. 본 수정으로 누락 액션도 트리거에 편입.
