## 배경
"MDM 기능 추가 가능 여부" 질의 → 타당성 검토 후 사용자가 A(MDM-lite) 선택, 스키마 변경 승인.
실제 디바이스 제어(원격 잠금/와이프/정책 푸시)는 Apple MDM/APNs·Android Enterprise·Intune 등
플랫폼 인프라가 필요해 별도 제품 영역 → 본 티켓은 자산의 등록·컴플라이언스·체크인 상태를
기록·가시화하는 경량 추적 계층(MDM-lite)만 구현.

## 설계
- DeviceCompliance(Asset 1:1): enrollmentStatus, complianceStatus, managed, 보안점검 6종(3-상태
  boolean: true/false/null=미점검), lastCheckinAt, enrolledAt, externalSource/externalDeviceId(B 연동 대비).
- DeviceCheckin(이력): 체크인 시각·결과·OS버전·IP·source.
- API: GET/PUT upsert(ADMIN), POST checkin(ADMIN, lastCheckinAt·status 갱신 + 이력 + AuditLog).
- PUT 부분수정: 보낸 필드만 반영(undefined=skip, null=clear). enrolledAt 은 첫 ENROLLED 전환 시 기록.
- 대시보드: 서버 컴포넌트 집계 + 클라이언트 렌더(useTranslation). lastCheckinAt 30일 초과/없음 = stale.
- 마이그레이션: 로컬 DB 미연결 → `prisma migrate diff`(스키마↔스키마)로 SQL 생성. 런타임은 db push.

## 후속(미구현)
- 미준수/장기 미체크인 알림(NotificationLog 연동) — 알림 스케줄러 통합 필요, 별도 티켓.
- 외부 MDM(Jamf/Intune) 동기화(옵션 B) — externalSource 필드로 토대 마련됨.
- jailbrokenRooted 등 점검 항목 자동 수집(에이전트/API) — 현재는 수동 입력.
