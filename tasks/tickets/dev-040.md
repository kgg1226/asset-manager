## 진단 (백로그 D3 — 정정)
백로그는 "할당 UI 3곳 전부 평면 select"였으나 실제 코드 확인 결과 **평면 select 는 보안 조직도 1곳뿐**.
나머지 2곳은 이미 검색 UI 보유:
- 하드웨어 상세(app/hardware/[id]/page.tsx 545~569): 검색 input + 버튼 리스트(/api/employees?q=).
- 라이선스 할당(app/licenses/assign-button.tsx 143~184): 검색 input + 체크박스 다중선택.
→ 동작하는 UI 교체는 회귀 리스크뿐이라 제외(최소 영향 원칙). 서버 검색 API 는 이미 완비.

## 구현
- EmployeePicker: 서버 검색(status=ACTIVE&search=, 이름·부서·이메일 contains) 250ms 디바운스,
  결과 최대 50건, 단일 선택, 미배정 옵션, 바깥 클릭 닫기. 좁은 카드/모달 호환(컴팩트).
- 보안 조직도: editEmployeeId 인라인 편집의 select 를 EmployeePicker 로 교체.
  기존 employees 배열은 initialLabel(현재 배정자 표시)에만 사용.

## 후속 (선택)
- 하드웨어/라이선스 피커를 EmployeePicker 로 통일(조직/부서 필터 공통화) — 기능 동일, 일관성 목적.
- 조직도 직접 할당(D3-B): 노드에 멤버 표시 + 할당 버튼.

## 검증
- npm run build 통과, 신규+수정 파일 lint 클린(에러·경고 0).

## 추가 (통일 — 사용자 요청 "하드웨어/라이선스 피커 통일")
- EmployeePicker 확장: items(부모 제공 목록→클라 검색) 모드 + 다중 선택(multi/selectedIds/

- 하드웨어 상세: 검색 input+버튼 리스트 → EmployeePicker(단일). employees/employeeSearch/

- 라이선스 할당: 검색 input+available useMemo+체크박스 리스트 → EmployeePicker(multi).
  좌석 제한(maxSelect)·배정자 제외(excludeIds) 보존. 부모(licenses-content/page) 무수정.
→ 직원 선택 UI 3곳이 단일 공용 컴포넌트로 통일.
