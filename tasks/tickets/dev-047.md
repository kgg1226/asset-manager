## 배경 (issues-backlog-2026-06b P2-c, 설계: dev-pii-items-design.md)
PII 항목이 엣지(AssetLink.piiItems)에만 있어 흐름 없는 자산의 수집 항목·보유기간 추적 불가.
구조 기획 + 적합성 시뮬레이션(회원 서비스 시나리오)으로 ISMS-P 흐름표 6컬럼 충족 검증 후 진행
(사용자 방침: 사전 승인 대신 시뮬레이션 후 진행).

## 구현
- lib/pii-items.ts: 21개 항목 카탈로그(코드·분류 GENERAL/UNIQUE_ID/SENSITIVE/FINANCE/CREDENTIAL·
  한국어 라벨·색상). 엣지 piiItems 와 공유 가능한 단일 출처.
- AssetPiiItem(additive·무위험): assetId FK(Cascade), itemKey, category, lifecycleStages(JSON 복수),
  legalBasis, retentionPeriod, destructionMethod, note. @@unique([assetId,itemKey]). Asset.piiStage 공존.
- API: POST createMany(카탈로그 검증+category 서버 도출), PUT 전체 교체(deleteMany+createMany),
  GET include. itemKey 미카탈로그 항목은 필터 제외.
- PiiItemsEditor: 분류별 색 칩 다중선택 + 선택 항목별 보유기간·법적근거 인라인 입력.
- 폼 8곳: PiiStageSelect 아래 배치, payload piiItems, 편집 폼 기존값 로드.
