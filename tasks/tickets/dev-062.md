## 배경 — 사용자 보고 (v1.61.0 스크린샷)
① 페이지를 나갔다 재진입하면 배치했던 자산이 전부 비어 다시 추가해야 함.
② 개인정보 흐름도 뷰는 팔레트가 "등록된 자산이 없습니다"라 할 수 있는 것이 없음.

## 원인
① **stale 워크스페이스 로드**: loadWorkspace 가 갤러리 목록(마운트 시점 스냅샷)의 객체를 그대로 열었다.
   세션 중 autosave 로 저장된 위치·소속이 그 객체엔 없어 캔버스가 비고, 이후 병합 저장(dev-059)이
   stale(빈) 저장본과 합쳐져 **서버의 정상 데이터를 되덮었다**(영구 소실 경로). flushSave 는
   activeWorkspace ref 만 갱신하고 갤러리 목록은 갱신하지 않던 간극.
② **뷰-필터된 자산 유니버스**: 비-all 뷰 API 는 뷰-연결 자산만 반환 → setAllAssets(팔레트)가 비고,
   수동 배치 자산도 fetchedAllAssets.filter 에서 탈락. 또 비-all 노드 배치가 connected+수동배치만
   포함하고 저장된 배치(savedIds)를 무시해 재진입 시 비-all 뷰에서 자산이 사라졌다.

## 수정
- loadWorkspace: 열기 전에 GET /api/asset-map/views 로 **서버 최신본**을 받아 적용(+갤러리 목록 동기화).
  조회 실패 시 기존 객체 폴백.
- fetchGraph: 비-all 뷰에서 view=all 을 병렬 fetch 해 **노드/팔레트 유니버스는 항상 전체 자산**,
  뷰 파라미터는 엣지 필터로만 사용.
- 비-all 노드 배치에 savedPlacedIds(저장된 위치 키) 포함 — "페이지=자산 배치, 뷰=엣지 필터" 일관화.
  PII 뷰에서 배치 자산이 단계 밴드 레이아웃으로 표시되고 연결 작성 가능.
