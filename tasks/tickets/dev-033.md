  - npm run build / lint 통과

## 배경 (백로그 F1)
"동기화 버튼을 누르지 않더라도 실시간 동기화" 요구. 진단 결과 cron 라우트(BE-056)·CRON_SECRET
인증·무료 API 폴백까지 전부 구현돼 있으나 **트리거(스케줄러)가 어느 배포 경로에도 연결돼 있지 않아**
버튼을 누르지 않으면 영원히 갱신되지 않는 상태였다.

## 구현
- 동기화 본체를 lib/exchange-rate-sync.ts 로 추출(syncExchangeRates) — cron 라우트와 lazy 경로가 공유.
  OXR fetch 에 타임아웃(5s) 추가(기존엔 무한 대기 가능).
- ensureTodayRates(): 오늘 데이터 없으면 그 자리에서 동기화. 실패 시 1시간 스로틀(인메모리,
  단일 인스턴스 배포 전제) 후 재시도 — 외부 장애가 조회 latency 로 전파되는 것 차단.
- 연결 지점: GET /api/exchange-rates(폼 자동 적용·교차환율), GET /api/admin/exchange-rates(관리 화면).
- "실시간"의 해석: 환율 데이터 소스 자체가 일 단위 갱신 → 일 1회 + 첫 사용 시점 자동이 실질 최적.
  더 촘촘한 갱신이 필요하면 EC2 crontab 1줄(기존 cron 라우트)로 보강 가능(배포 문서 참고).
