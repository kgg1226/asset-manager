## 배경
자산지도 연결 누적·중복 문제(사용자 보고)의 서버부. dev-055(다중선택 일괄삭제)·dev-056(중복 정리 버튼)이 소비.

## 설계 결정
- 중복 키 = source/target(자산·외부) + linkType + **direction** — REVERSE/BI/CONDITIONAL 은 의미가 달라
  direction 다른 연결을 지우면 기능 삭제(파괴 방지 규칙 6). A→B 와 B→A 도 별개(보수적).
- 생존자 = 메타점수(piiItems 항목수×10 + 비어있지 않은 메타필드 수) → updatedAt 최신 → id 최대.
  PII 증적(dev-050/051)이 달린 연결을 최우선 보존.
- 권한 = 인증 사용자(기존 단건 DELETE 와 동일 수준 — requireAdmin 상향은 비대칭이라 배제).
- ids 검증 Number.isInteger(dev-049 #13 NaN 교훈), 배치 상한 200, 미존재 ID 는 404 대신 notFound[] 보고(화면 stale 대비).
- dedup 후 all 뷰 visibleEdgeIds 고아 ID 는 무해(fetchGraph 필터 매칭 실패) — 다음 autosave 가 자연 재작성.
