## 근본 원인 (후보군 로드맵 A3)
섹션 노드 data 키가 두 갈래로 갈림:
- 런타임(생성 handleAddSection·렌더 SectionNodeComponent·편집 모달): label / description / sectionColor
- 직렬화(flushSave 2곳·복원): sectionName / sectionDescription / sectionColor
→ 저장 시 n.data.sectionName(undefined) 을 읽어 항상 빈값 저장 → 복원 시 빈값 →
   렌더가 data.label 읽으면 빈 헤더. 섹션 이름·설명 round-trip 유실(데이터 유실 버그).

## 수정 (직렬화/복원을 런타임 키에 맞춤 — 화면 동작 코드 무변경, 회귀 최소)
- flushSave(2195, 2282): n.data.sectionName/sectionDescription → n.data.label/description 읽기
- 복원(2849): 노드 data 생성 시 sectionName/sectionDescription → label/description
- SectionDataItem JSON 형식(name/color/description)은 유지(외부 저장 형식).
- 생성·렌더·편집 코드는 0 변경(이미 label/description 정상 동작).

## 참고
기존 저장 데이터의 섹션 name 은 이미 빈값으로 유실됨(소급 복구 불가) — 본 수정 후
사용자가 섹션 이름을 재입력하면 정상 보존.
