## 배경 (사용자 제안 이미지)
"관계 그래프 + 노드를 클릭해 따라가는 탐색" 제품 이미지를 보고 적용 가능성 질의.
진단: 시각화(React Flow)·연결 목록(사이드패널)·통합검색은 이미 있고, 빠진 것은
"검색 → 그래프 진입 → 이웃 확장"이라는 **탐색 동선**. 그중 A단계(진입)를 먼저 구현.

주의: 원본 이미지는 문서에서 AI가 관계를 추출하는 제품이나, 본 시스템은 연결이 이미 명시
데이터라 AI 추출이 불필요하다. 강점(자산 인벤토리·PII 증적·CIA)에 맞춰 번역해 적용.

## 구현
- init(): ?focus=<id> 파싱(정규식 정수 검증) → nodePositions 에 해당 자산이 있는 페이지 탐색
  (isDefault 우선 → 최근 접근 순) → 그 페이지를 열고 pendingFocusId 설정. localStorage 복원보다
  우선(사용자가 방금 지정한 의도). 미배치면 focusNotFound 배너.
- 포커스 effect: loading 종료 + 노드 존재 확인 후 1회 실행, pendingFocusId 즉시 비워 재발화 차단.
  복원 effect(setViewport)와 경합하지 않도록 rAF 로 미루고 setCenter(zoom 1.2)+노드 선택+사이드패널 오픈.
- 자산 상세 4종·통합검색 결과에 진입 버튼(Network 아이콘). 검색은 기존 행 클릭(상세)을 유지하고
  별도 아이콘 버튼으로 분리 — 기존 동선 파괴 없음.

## 후속 (B·C 단계, 사용자 확인 후)
- B: 노드 우클릭 "연결된 자산 펼치기"(1-hop 이웃 확장)
- C: 영향 범위 패널(N-hop 도달 자산·PII 항목·수탁사 집계)
