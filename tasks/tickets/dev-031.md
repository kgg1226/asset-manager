## 문제
하드웨어 페이지 Excel 버튼이 type=HARDWARE 를 넘기지만, export/all 라우트가 자산 쿼리에만
필터를 적용하고 워크북에는 항상 Licenses(소프트웨어)·Assets(범용)·Employees·Cost Summary
4개 시트를 모두 포함 → 하드웨어에서 받아도 첫 시트가 소프트웨어라 "소프트웨어만 내려받기 됨"
으로 인지. Assets 시트도 범용 컬럼이라 하드웨어 실사에 필요한 컬럼(호스트명/OS/일련번호 등) 부재.

## 구현
- type 파라미터 화이트리스트 검증(잘못된 값 400).
- type 있으면: 해당 유형 자산만 조회(hardwareDetail/cloudDetail/domainDetail/contractDetail
  include) → getTypedSheet(type) 단일 출처에서 시트 정의(한국어 헤더) → Excel/CSV 공용.
  파일명도 유형별(hardware-export-YYYY-MM-DD.xlsx).
- HARDWARE 컬럼은 사용자 지정 그대로. 등급 = lib/cia 의 getCiaOverallGrade(1등급=최상).
- CLOUD/DOMAIN_SSL/CONTRACT/기타 도 각 디테일 모델 기반 전용 컬럼.
- type 없으면 기존 전체 워크북 그대로(기능 보존).

## 알려진 한계
- "담당자"·"관리자"는 현재 데이터 모델에 없는 필드(Asset/HardwareDetail/import 템플릿 모두 부재)
  → 헤더는 유지하되 빈 컬럼(실사 수기 기입용). 저장이 필요하면 스키마 필드 추가(승인 필요) 후속.
