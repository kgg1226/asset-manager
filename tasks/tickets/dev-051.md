## 배경 — 결점 체크 #8 "노드/엣지 PII 어휘 분기"

노드는 PII_ITEM_CATALOG 코드, 엣지 piiItems 는 LinkModal 자유텍스트(split ",")로 어휘가 갈라져
조인·집계·증적 통합이 불가능했다. lib/pii-items.ts 주석이 약속한 "단일 출처 공유"를 코드로 닫는다.

### 변경 (asset-map-content.tsx 단일 파일)
- 모듈 헬퍼 추가: parseEdgePiiItems(JSON 배열/레거시 콤마 → 배열), edgePiiItemsLabel(코드→라벨/원문 유지).
- LinkModal: piiItemsText(자유텍스트 state) → selectedPiiItems(코드 배열). 입력 UI 를 카탈로그 칩
  다중선택으로 교체(PII_ITEM_KEYS/PII_CATEGORY_STYLE — PiiItemsEditor 와 동일 스타일). 저장은 코드 배열.
- EdgeDetailModal 표시: 자유텍스트 <p> → 칩 리스트. isPiiItemKey 로 코드는 분류색 칩+라벨,
  레거시 자유텍스트는 붉은 칩+원문(파괴 금지).
- PII 뷰 엣지 라벨: 원문 그대로 → edgePiiItemsLabel(코드 라벨화, 레거시 원문 유지).

### 하위호환 (파괴적 마이그레이션 없음)
- 기존 DB 자유텍스트("이메일,이름")와 신규 코드("EMAIL","NAME")가 같은 JSON 배열 컬럼에 공존.
- 모든 표시/라벨 경로가 isPiiItemKey 단일 분기로 코드↔원문 이원 처리.
- 신규 저장은 항상 코드 → 레거시 행은 재저장 전까지 원문 유지, 점진 수렴.
