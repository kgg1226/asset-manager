## 배경
PII 기능이 한국어 하드코딩(58개 문자열)이라 다국어 미지원이었다. 결점 체크 백로그의 "PII 다국어".

## 설계 (멀티에이전트 매핑+적대 검증)
- **신규 `pii` 네임스페이스** — 항목 라벨 21 + 에디터/리포트 UI 8 = 29키 × 6로케일.
- **단계 라벨은 기존 `assetMap.pii*` 재사용**(중복 신설 금지) — piiStageLabelI18n 이 내부에서 참조.
- **카테고리 라벨은 미렌더**(색상 bg/text만 사용) → i18n 제외, lib 유지.
- **헬퍼는 t 주입형으로 추가**(piiItemLabelI18n/piiStageLabelI18n), 기존 piiItemLabel/PII_STAGE_LABEL 는
  서버 export 용으로 **보존**(삭제 금지). 서버는 locale 획득 수단이 없어 한국어 폴백 — 국내 증적 특성상 적절.
- 라벨이 함수뿐 아니라 직접 `.label` 접근으로도 소비되던 지점(editor·asset-map 4곳)을 전부 헬퍼 경유로 통일.
