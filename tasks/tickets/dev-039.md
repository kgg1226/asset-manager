## 배경 (백로그 D1-A)
"왜 4개 카드가 고정인가" 답변: pii 뷰 전용 스윔레인 행 헤더. 격차 ③ Asset.piiStage 필드는
있는데 설정 UI 가 없어 흐름도가 자산 이름 패턴 추측(web→수집 등 엉성)에 의존 → 증적 신뢰성 약함.
격차 ① pii 뷰는 영역(섹션)과 공존 불가(getPiiLifecycleLayout 이 section 노드를 필터로 버림).

## 구현
Part 1 (주가치 — piiStage 지정):
- lib/pii-stage.ts: PII_STAGES(COLLECTION/STORAGE/USAGE_PROVISION/DESTRUCTION) + isPiiStage.
- PiiStageSelect: 미지정=null 포함 셀렉트, 단계명은 기존 i18n 재사용.
- API POST/PUT: isPiiStage 검증 후 수용(PUT 은 명시 null 허용=단계 해제). GET 은 기존 포함.
- 폼 8곳: ClassificationSelect 아래 배치, 편집 폼은 기존값 로드.

Part 2 (섹션 공존 — 최소):
- getPiiLifecycleLayout: sectionNodes 추출 + 반환 맨 앞(배경) 포함. 저장 위치 유지.
- 단계 행 자산은 parentId/extent 해제(절대좌표 — 섹션 되살림에 따른 필수 처리).
- 영역×단계 완전 조합 배치는 D1-B(단계 카드 팔레트화)에서.

## 검증
