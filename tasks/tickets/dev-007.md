SCOPE:
  - app/hardware/page.tsx
  - colors_and_type.css (신규 — 디자인 토큰 레퍼런스)
  - preview/*.html (신규 — 컴포넌트 비주얼 레퍼런스 21종)
  - ui_kits/asset-manager/{index.html,README.md} (신규 — 인터랙티브 UI Kit)
  - assets/logo-placeholder.svg (신규)
  - fonts/geist-mono-latin.woff2 (신규 — 디자인 킷용 표준 사본)
ACCEPTANCE:
  - 하드웨어 페이지 감가상각 카드가 좁은 폭(grid-cols-2)과 넓은 폭(md:grid-cols-4)에서 모두 깨지지 않음
  - 큰 금액은 백만원 단위로 압축 표기되고 `title` 속성에 원본 금액이 보존됨 (formatCostMillions)
  - npm run build 통과
  - 디자인 레퍼런스 파일은 Next.js 빌드/번들에 포함되지 않음 (app/, public/ 외 위치)
COMPLEXITY: low
DEPLOY_FLAG: false
SECURITY_FLAG: false

## 배경
별도 작업으로 추출된 클로드 디자인 시스템 (토큰 + 컴포넌트 비주얼 + UI Kit)을 레포에
참조용으로 추가. 동시에 하드웨어 페이지의 감가상각 요약 카드에서 큰 금액 문자열이
좁은 화면에서 카드를 깨뜨리던 문제를 해결.

## 변경 요약
- `formatCostMillions(cost, unit)` 추가 → 큰 금액을 백만원 단위로 압축
- 감가상각 그리드를 `grid-cols-2 md:grid-cols-4` 로 반응형화
- 각 셀에 `min-w-0`, 값에 `truncate tabular-nums` + 원본 금액을 `title`에 보존
- 카운트 칩 줄을 `flex-wrap gap-1.5` 로 변경 — 좁은 폭에서 줄바꿈 허용
- 디자인 레퍼런스 디렉토리 추가 (런타임 영향 없음)

## 비-적용 결정
- `colors_and_type.css` 의 토큰을 `app/globals.css` 에 인라인하지 않음
  → 현재 앱은 stock Tailwind 클래스 사용 중이며, CSS 변수 도입은 별도 티켓 권장
- `assets/{file,globe,next,vercel,window}.svg` 와 `fonts/geist-mono-latin.woff2` 는
  `public/` 의 사본과 중복이지만 디자인 킷의 자기충족(self-contained) 동작을 위해 유지
  → 추후 디자인 킷을 `docs/design/` 으로 이전할 때 재정리 권장 (roadmap)
