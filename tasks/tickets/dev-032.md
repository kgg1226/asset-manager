## 근본 원인 (백로그 B1 — 멀티에이전트 진단으로 확정)
dev-021(976e416)이 flushSave 에 추가한 `setActiveWorkspace`(매 자동저장마다 새 객체 참조)가
fetchGraph effect(의존성=activeWorkspace **객체**)를 재발화시키는 자기 파괴 루프:
사용자 조작 → markDirty → flushSave → setActiveWorkspace(새 참조) → fetchGraph 전체 재로드
→ dagre 재배치 → 복원 effect 가 stale 위치·뷰포트 재적용 (2~3초 주기 "계속 초기화").

자산 추가 소실도 같은 뿌리: flushSave 가 로컬 ws 에 edgeVisibility 만 갱신하고 nodePositions 는
갱신하지 않아, 직후 fetchGraph 가 stale ID 필터로 방금 추가한 자산을 캔버스에서 탈락시키고
다음 flushSave 가 그 상태로 서버를 덮어써 영구 소실 (dev-020 보호장치의 회귀).

## 수정 (5개 edit, 단일 파일)
1. restoringViewportRef 신설 — 복원 중 setViewport 가 onMoveEnd 를 자체 발화시키는 루프 가드
2. flushSave: 로컬 ws(ref+state)를 방금 저장한 전체 상태(nodePositions/sectionData/viewport/
   edgeVisibility)로 병합 — stale 필터·stale 복원 차단. state 도 갱신하는 이유: 매 렌더마다
   ref 가 state 로 동기화되므로 ref 만 갱신하면 다음 렌더에 stale 값이 되살아남.
3. fetchGraph effect 의존성: activeWorkspace 객체 → activeWorkspace?.id (저장이 재로드를 못 일으킴.
   워크스페이스 전환(id 변경)·뷰 전환(fetchGraph 재생성)은 기존대로 재로드)
4. 복원 effect: id 키잉 + activeWorkspaceRef 에서 최신 상태 읽기 + setViewport 복원 가드
5. onMoveEnd: 복원 중 발화는 무시 (사용자 조작이 아님)
