## 배경
연결 삭제가 복잡하고(더블클릭→모달→버튼), 다중 조작 불가, 수정 시 기존 값이 유실되던 문제(사용자 보고 + 매핑 ③).

## 변경
- **다중선택**: onSelectionChange 에 edges 추가 → multiSelectedEdgeIds. 기존 selectionOnDrag/Partial 설정 그대로.
- **우클릭 메뉴**: MapContextMenu(신규, 서브메뉴 지원) + onEdgeContextMenu/onPaneContextMenu.
  엣지 단건(수정/방향반전 UNI↔REVERSE/유형·방향 변경/삭제), 다중(N개 유형·방향 변경/삭제), 빈 캔버스(연결 추가).
  우클릭 팬(panOnDrag=[2]) 직후 contextmenu 오발화는 onMoveEnd 타임스탬프 150ms 가드.
- **키보드 삭제**: deleteKeyCode Delete/Backspace + onBeforeDelete(엣지만 허용·노드 차단·confirm) + onEdgesDelete(bulk 반영, 실패 시 fetchGraph 롤백).
- **삭제 단순화**: handleDeleteEdges(bulk-delete API 1콜 + fetchGraph 1회 — dev-045 핸들 재분배 회귀 방지). 기존 handleDeleteEdge 는 단건 래퍼로 보존.
- **편집 값보존**: LinkModal initialData(기존 엣지 data 파싱 — dataTypes/piiItems JSON, 레거시 자유텍스트 piiItems 는 숨은 채 보존·병합) + 편집 시 PUT(handleUpdateLink). EdgeDetailModal '수정'과 우클릭 '수정' 모두 적용.
- **선택 강조**: globals.css 에 selected 엣지 stroke 3.5px + glow.
- i18n: deleteSelected/flipDirection/changeType/changeDirection 4키 × 6로케일 (editLink/deleteLink 는 기존 키 재사용 — 중복 선언 시도를 tsc 게이트가 잡음).
