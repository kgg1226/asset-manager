# Phase 5 — UX 개선 + 버그 수정 + 자산관리 고도화

> 작성: 기획 세션 (2026-03-15)
> 대상: Ralph loop 에이전트
> 목표: 실사용 피드백 기반 UX 개선, 버그 수정, 자산관리 완성도 향상

---

## 배경

Phase 1~4까지 핵심 기능이 구현되었으나, 실사용 과정에서 UX 불편사항·버그·기능 미비가 다수 발견됨.
이번 Phase 5는 **"완성도"** 에 집중하여 실무 투입 가능한 수준으로 끌어올린다.

---

## 요구사항 목록

### R-001. 비밀번호 수동 변경

**현황**: `mustChangePassword` 강제 변경 UI는 존재하나, 사용자가 **자발적으로** 비밀번호를 변경하는 UI가 없음.
`POST /api/auth/change-password` API는 구현 완료.

**요구사항**:
- ADMIN / USER 모두 본인 비밀번호를 변경할 수 있는 UI 필요
- 현재 비밀번호 확인 → 새 비밀번호 입력 → 확인 입력 3단계
- 비밀번호 최소 8자, 영문+숫자 필수 (보안 가이드라인 준수)
- 변경 성공 시 토스트 알림, 실패 시 에러 표시

**위치**: `/settings/profile` 내 비밀번호 변경 섹션 (또는 별도 모달)

---

### R-002. 내 프로필 페이지

**현황**: 로그인한 사용자가 본인 정보를 확인·수정할 수 있는 페이지 없음.

**요구사항**:
- 경로: `/settings/profile`
- 표시 정보: 사용자명, 역할, 계정 생성일, 마지막 로그인
- **ADMIN**: 사용자명 변경 가능 (중복 체크)
- **USER**: 비밀번호만 변경 가능 (다른 정보는 읽기 전용)
- 비밀번호 변경은 R-001의 UI 재사용

**API**:
- `GET /api/auth/me` — 본인 정보 조회 (세션 기반)
- `PUT /api/auth/me` — 본인 정보 수정 (ADMIN: username 변경 가능 / USER: 비밀번호만)

---

### R-003. 라이선스 "No Key" 표시 제거

**현황**: 라이선스 키가 없는 항목에 "No Key" 텍스트가 표시됨. 불필요한 정보.

**요구사항**:
- `key` 필드가 null이거나 빈 문자열일 때 "No Key" 텍스트를 표시하지 않음
- 해당 영역은 빈 칸으로 두거나, 키 필드 자체를 숨김
- 영향 범위: 라이선스 목록, 라이선스 상세, 라이선스 수정 폼

---

### R-004. 조직원 부서 필드 → 조직/하위조직 선택으로 대체

**현황**: Employee에 `department` (자유 텍스트) 필드가 있으나, 이미 `companyId` + `orgUnitId`로 조직 구조가 관리됨. `department` 텍스트 필드는 중복이며 조직도와 불일치 위험 존재.

**요구사항**:
- 조직원 등록/수정 폼에서 `department` 텍스트 입력란 제거
- 대신 **회사(Company) 선택 → 조직(OrgUnit) 계층 선택** 드롭다운으로 대체
  - 1단계: 회사 선택 (필수)
  - 2단계: 조직 선택 (선택, 계층 트리 드롭다운)
- `department` 필드는 DB에 유지하되, 선택한 조직의 이름을 자동 저장 (하위호환)
- 조직원 목록/상세에서도 부서 대신 **회사 > 조직** 경로 표시

---

### R-005. 하드웨어 장비 유형별 동적 입력 필드

**현황**: 모든 하드웨어 자산이 동일한 폼 필드를 사용. 장비 유형(Laptop/Desktop/Server/Network/Mobile 등)에 따라 필요한 정보가 다름.

**요구사항**:

#### 공통 필드 (모든 유형)
- 자산명, 상태, 제조사, 모델, 시리얼넘버, 자산태그, 설명

#### 유형별 추가 필드

| 유형 | 추가 필드 |
|---|---|
| **Laptop / Desktop** | OS, OS 버전, 호스트명, MAC Address, CPU, RAM(GB), 저장장치(GB), 내용연수(년) |
| **Server** | OS, OS 버전, 호스트명, IP Address, MAC Address, CPU, RAM(GB), 저장장치(GB), 위치(서버실/IDC) |
| **Network** (라우터/스위치/AP) | 호스트명, IP Address, MAC Address, 포트 수, 위치 |
| **Mobile** (태블릿/폰) | OS, OS 버전, IMEI, 전화번호 |
| **Peripheral** (모니터/키보드/마우스) | 연결 방식(USB/BT/기타), 해상도(모니터용) |
| **Other** | 비고 (자유 텍스트) |

#### Laptop/Desktop 수명 자동 만료일
- 관리자가 **설정 > 하드웨어 수명 주기**에서 유형별 기본 내용연수 설정
  - 예: Laptop = 4년, Desktop = 5년
- 하드웨어 등록 시 `purchaseDate` + 내용연수 → `expiryDate` 자동 계산
- 개별 자산에서 내용연수 override 가능
- 만료일 도래 시 대시보드·알림에 표시

**DB 변경**:
- `HardwareDetail` 모델에 추가 필드: `cpu`, `ram`, `storage`, `imei`, `phoneNumber`, `portCount`, `connectionType`, `resolution`
- 신규 모델: `HardwareLifecycleSetting` (유형별 기본 내용연수 관리자 설정)

### R-006. 도메인/SSL 비용 주기 개선

**현황**: 도메인/SSL은 비용 주기가 1년/2년/3년 등 다양하나, 현재 `billingCycle`이 MONTHLY/ANNUAL/ONE_TIME만 지원.

**요구사항**:
- 도메인/SSL 전용 입력 필드 추가:
  - 도메인명 (예: example.com)
  - 등록 기관 (Registrar)
  - SSL 인증서 유형 (DV/OV/EV/와일드카드)
  - 발급 기관 (CA)
  - 만료일 → **갱신일**로 용어 변경 (도메인은 갱신 개념)
  - 비용 주기: MONTHLY / YEARLY / BIENNIAL(2년) / TRIENNIAL(3년) / CUSTOM
  - CUSTOM 선택 시 주기(개월 수) 직접 입력
- `expiryDate` 필드명은 유지하되, UI 라벨을 "갱신일"로 표시
- 갱신일 기준으로 월 환산 비용 자동 계산:
  - `monthlyCost = cost / (주기 개월 수)`

**DB 변경**:
- 신규 모델: `DomainDetail` (도메인/SSL 전용 상세 정보)
- `billingCycle` 확장 또는 별도 필드: `billingCycleMonths Int?` (커스텀 주기)

### R-007. PDF 보고서 내보내기 수정

**현황**: PDF 보고서 내보내기 시 한글 문자가 깨지고, 서버가 크래시됨.

**원인 추정**:
- 한글 폰트 미포함 (폐쇄망이라 외부 폰트 로딩 불가)
- PDF 라이브러리 메모리 초과 (EC2 RAM 2GB 제한)

**요구사항**:
- 한글 폰트를 프로젝트에 번들링 (NotoSansKR 등, woff2 파일)
- PDF 생성 라이브러리 교체 검토 (현재 사용 중인 라이브러리 확인 후 결정)
  - 옵션 A: `@react-pdf/renderer` (React 기반, 폰트 내장 용이)
  - 옵션 B: `pdfmake` (서버 사이드, 가벼움)
  - 옵션 C: `jsPDF` + `jspdf-autotable` (테이블 지원)
- 대용량 데이터 스트리밍 처리 (메모리 제한 2GB 고려)
- 폐쇄망 제약: 외부 CDN/폰트 URL 호출 금지

---

### R-008. 하드웨어 자산 등록 403 에러 수정

**현황**: 하드웨어 자산 등록(POST) 시 403 Forbidden 에러 발생.

**원인 추정**:
- `POST /api/assets` 라우트의 RBAC 검증 또는 인증 체크 문제
- `proxy.ts`에서 해당 경로가 차단되고 있을 가능성

**요구사항**:
- 403 에러 원인 파악 및 수정
- 인증된 사용자(ADMIN)가 자산을 등록할 수 있어야 함
- 수정 후 모든 자산 유형(SOFTWARE, CLOUD, HARDWARE, DOMAIN_SSL) 등록 테스트

### R-009. 이력 페이지 이름 통일

**현황**: 사이드바에는 "이력", 페이지 제목은 "감사 로그" 등 용어가 혼재.

**요구사항**:
- 모든 곳에서 **"변경 이력"** 으로 통일
  - 사이드바 메뉴명: "변경 이력"
  - 페이지 제목: "변경 이력"
  - 브라우저 탭: "변경 이력 | Asset Manager"
- 코드 변수명/API 경로(`/history`, `/api/history`)는 유지 (하위호환)

### R-010. 이력 유형/액션별 색상 구분

**현황**: 감사 로그 목록에서 모든 항목이 동일한 스타일로 표시.

**요구사항**:
- 액션별 색상 배지:
  - `CREATED` — 초록 (#22C55E)
  - `UPDATED` — 파랑 (#3B82F6)
  - `DELETED` — 빨강 (#EF4444)
  - `LOGIN` / `LOGOUT` — 회색 (#6B7280)
  - `STATUS_CHANGE` — 주황 (#F59E0B)
  - `ASSIGNED` / `RETURNED` — 보라 (#8B5CF6)
- 엔티티 유형별 아이콘 또는 라벨 색상:
  - LICENSE — 파랑
  - ASSET — 초록
  - EMPLOYEE / MEMBER — 주황
  - USER — 회색
  - ORG — 보라

### R-011. 환율 자동 갱신 (매일 9시) + 수동 트리거

**현황**: `POST /api/admin/exchange-rates/sync` API 존재하나, 자동 스케줄링 미구현. 환율 관리 UI(`/admin/exchange-rates`)도 존재.

**요구사항**:
- **자동**: 매일 오전 9:00 KST 환율 자동 갱신
  - `POST /api/cron/exchange-rates` 배치 API 신규 생성
  - 외부 API 호출 (폐쇄망이나 outbound는 허용)
  - OpenExchangeRates 또는 한국은행 API 사용
  - 실패 시 1시간 후 1회 재시도
  - 성공/실패 여부 AuditLog 기록
- **수동**: 관리자가 환율 관리 페이지에서 "지금 동기화" 버튼 클릭
  - 기존 `POST /api/admin/exchange-rates/sync` 활용
  - 동기화 중 로딩 스피너 + 결과 토스트
- 마지막 동기화 시각 표시 (환율 관리 페이지 상단)

## 기획 추가 — 완성도 향상 항목

> 사용자 요구에 명시되지 않았으나, 완벽한 자산관리 시스템을 위해 기획이 추가 식별한 항목.

### R-012. 자산 대량 등록 (CSV 임포트)

**현황**: 라이선스/조직원/그룹 CSV 임포트는 존재하나, 자산(Hardware/Cloud/Domain) 대량 등록 미지원.

**요구사항**:
- `/settings/import` 페이지에 "자산" 탭 추가
- CSV 템플릿 다운로드 기능
- 유형별 필수/선택 필드 검증
- 오류 행 표시 + 부분 성공 지원

### R-013. 자산 만료 알림 대시보드 위젯

**현황**: 만료 임박 자산 API(`GET /api/assets/expiring`)는 있으나, 대시보드에 위젯 미노출.

**요구사항**:
- 대시보드에 "만료 임박 자산" 카드 추가
- D-30 이내: 빨간 배지, D-90 이내: 주황 배지
- 클릭 시 해당 자산 상세 페이지로 이동

### R-014. 설정 페이지 구조화

**현황**: `/settings/groups`, `/settings/import`만 존재. 하드웨어 수명 주기, 프로필 등 설정 항목 증가 예정.

**요구사항**:
- `/settings` 레이아웃에 좌측 탭 네비게이션 추가:
  - 프로필 (R-002)
  - 그룹 관리 (기존)
  - CSV 가져오기 (기존)
  - 하드웨어 수명 주기 (R-005 연계)
- ADMIN 전용 탭 구분 (일반 사용자에게 숨김)

## 비기능 요구사항

### NFR-001. 응답 시간
- PDF 생성: 최대 30초 (타임아웃 설정)
- 환율 동기화: 최대 10초
- 페이지 로드: 3초 이내

### NFR-002. 메모리 제한
- PDF 생성 시 최대 512MB 메모리 사용 (EC2 2GB RAM 중)
- 스트리밍 방식으로 대용량 데이터 처리

### NFR-003. 에러 복원
- 환율 동기화 실패 시 이전 환율 유지
- PDF 생성 실패 시 사용자에게 명확한 에러 메시지 표시
